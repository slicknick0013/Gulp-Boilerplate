About This Website:

I created this website as a place to write down my current thoughts on design philosophy and implementation details that aren't specific to any single project I'm working on. Mostly, it's a place to communicate with myself about things I don't want to forget, and a place to house my Résumé. It also serves as a place for me to do proof of concepts and to develop UI without a lot of overhead. With only a few modifications for this site (NickPruett.com) this is the basic boilerplate from which I begin. I start from a very neutral position and then work to build a specific brand.

Website Details:

This website uses a ruby framework called Susy for the grid layout and much of the responsive design elements. Take a second to view the site on your phone or shrink your browser page to experience the various breakpoints. The menu at the top of the page is at the bottom of the HTML hierarchy and uses CSS to position it where I want it. On a smaller device it is a clickable entity. The server is ran on my home computer with my router forwarding traffic to a specific machine running node commands. I use Gulp.js to simplify the build process for SCSS files and to automate minifying and concatenating files. Everything that is on here is ran without an app server so could be used in any project regardless of middle tier (Java, C#, PHP) etc. The URLs are search engine friendly using index files to be served up to generic URLs. IE: NickPruett.com/resume instead of NickPruett.com/resume.html

Résumé:

The Résumé is the only thing that doesn't follow the standards for the rest of the site. It is built for print and was created long before this. It is an orphaned page so that I don't have to create something that strips out the nav/header/footer etc when it needs to be printed or converted to PDF.

Styleguide:

The Styleguide is a place to present UI elements apart from specific implementation. The code that handles individual details is included below each section for code review and to verify standards once implemented by different people. The styleguide for my site has not really been altered from basic styles. If changes were wanted I could make a change and publish it publicly and stakeholders could discuss in real-time any thoughts on changes.

Modules:

Modules is a place for me to put javascript modules that help to enhance the UI. I will be migrating some of the existing stuff I use here in the future.
